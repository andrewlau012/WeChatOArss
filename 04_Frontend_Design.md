# 前端页面设计 (Frontend Design)

## 1. 页面结构图

```text
/ (Root) --> 首页 (Home)
├── 左侧栏：公众号列表 (支持搜索/添加/200-300个滚动展示)
└── 右侧栏：文章列表 (时间倒序/Web跳转/RSS链接)
/settings       # 系统设置 (含账号管理、全局配置)
/logs           # 抓取日志
```

## 2. 关键页面详情

### 2.1 首页 (Home) - 双栏布局
**左侧栏：公众号列表 (Feed List)**
- **头部**：
  - 搜索框：输入关键词过滤列表。
  - [+] 添加按钮：点击弹出模态框，支持 **“关键词搜索添加”** 和 **“链接解析添加”**。
- **列表区域**：
  - 虚拟滚动列表 (Virtual Scroll)，支持流畅展示 200-300+ 个公众号。
  - 每个Item显示：头像、名称、未读数(可选)。
  - 选中状态高亮。
- **底部**：
  - “全部文章” 选项（默认选中）。

**右侧栏：文章列表 (Article List)**
- **头部**：
  - 当前选中公众号名称/“全部文章”。
  - [RSS图标]：点击复制当前列表的 RSS 订阅链接。
  - [刷新]：手动触发当前公众号的抓取。
- **列表内容**：
  - 按发布时间倒序排列。
  - 卡片式展示：标题、摘要、封面图(可选)、发布时间、公众号来源。
  - **点击行为**：点击卡片直接跳转到 Web 浏览器打开原文链接（支持 `target="_blank"`）。
- **分页/加载**：滚动到底部自动加载更多。

### 2.2 设置页 (Settings)
包含以下选项卡：
1.  **账号管理**：
    - 列表展示已登录的微信读书账号。
    - **状态检测**：显示账号状态（正常/风控中/失效）。
    - [扫码添加]：新增账号入口。
2.  **抓取配置**：
    - 定时任务表达式。
    - 数据保留策略：默认保留 60 天（可编辑）。
    - RSS输出条数：默认 20 条（可编辑）。
3.  **系统信息**：版本号、数据库大小等。

## 3. 交互体验要求
- **响应式**：适配桌面端与移动端浏览器。
- **实时反馈**：扫码登录时，前端应轮询状态并在扫码成功后立即跳转。
- **错误提示**：若Cookie失效，顶部全局红条警示。
